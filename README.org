* Overview
This Emacs package, "Salesforce Development Tools for Emacs," provides a comprehensive suite of tools to enhance Salesforce development within the Emacs environment. It is designed to streamline the development process by integrating various Salesforce functionalities directly into Emacs, making it easier for developers to work with Salesforce technologies.

Key features include:

- **Apex Mode**: Provides a comprehensive environment for developing Salesforce Apex code with syntax highlighting, code completion, and integration with Salesforce DX.
- **SOQL Mode**: Provides a dedicated environment for writing and executing Salesforce Object Query Language (SOQL) queries. It includes syntax highlighting, auto-completion, and result visualization. The mode integrates with Eglot and LSP-bridge for enhanced code intelligence and navigation, and supports company-mode for intelligent code completion.
- **Visualforce Mode**: Provides a specialized environment for Visualforce page development with tree-sitter-based syntax highlighting for HTML, JavaScript, and CSS. It includes LSP integration for enhanced code intelligence and navigation, and supports auto-completion and code folding.
- **LWC Mode**: Offers support for Lightning Web Components (LWC) development with tree-sitter-based syntax highlighting for JavaScript and HTML files. It includes LSP integration for enhanced code intelligence, auto-completion, and navigation. The mode supports both Eglot and LSP-bridge for language server features.
- **LSP Support**: Integrates with Language Server Protocol (LSP) for enhanced code intelligence and navigation.
- **Debugging Tools**: Includes support for the Debug Adapter Protocol (DAP) to debug Apex code.
- **AI Integration**: Offers AI-assisted code commenting and documentation generation.
- **Folding Support**: Allows code folding for better code organization and readability.

* Installation
** Doom Emacs
#+begin_src emacs-lisp
(use-package! salesforce-minor-mode
   :recipe (:host github :repo "furucrm-minhtan/salesforce-minor-mode" :files ("*.el")))
#+end_src
** Vanilla Emacs
Clone the repository and add to your load path:
#+begin_src emacs-lisp
(add-to-list 'load-path "/path/to/salesforce-minor-mode")
(require 'salesforce-minor-mode)
#+end_src
* Key Bindings
The following key bindings are available in the Salesforce minor mode:

** Org Features
- `TAB`: Switch Org
- `r`: Retrieve Metadata
- `d`: Push Metadata
- `n`: List All Orgs
- `m`: List All Devhubs
- `p`: Diff File
- `;`: Execute Apex Code
- `.`: Open Org

** Resource Features
- `M-o o`: Org Features
- `M-o r`: Resource Features
- `M-o N`: Notes
- `M-o A`: Authorize Org
  
* Features
The "Salesforce Development Tools for Emacs" package offers a range of features to enhance Salesforce development, including:

** Apex Mode
Apex mode is a specialized environment within the "Salesforce Development Tools for Emacs" package, designed to facilitate the development of Salesforce Apex code. It includes:

- **Syntax Highlighting**: Provides clear and distinct syntax highlighting for Apex code, SOQL statements, and embedded expressions.
- **Code Completion**: Utilizes tree-sitter and company-mode for intelligent code completion and suggestions.
- **LSP Integration**: Supports both Eglot and LSP-bridge for language server features like code navigation and refactoring.
- **Debugging**: Integrates with the Debug Adapter Protocol for replay debugging of Apex code.
- **AI-Assisted Documentation**: Generates comments and documentation using AI, enhancing code readability and maintainability.
- **Code Folding**: Supports folding of code blocks for improved code management and navigation.

** Visualforce Mode
Visualforce mode is a specialized environment within the "Salesforce Development Tools for Emacs" package, designed to facilitate the development of Visualforce pages. It includes:

- **Syntax Highlighting**: Provides clear and distinct syntax highlighting for HTML, JavaScript, and CSS within Visualforce pages using tree-sitter.
- **Code Completion**: Utilizes company-mode for intelligent code completion and suggestions, enhancing productivity.
- **LSP Integration**: Supports both Eglot and LSP-bridge for language server features, offering enhanced code navigation and refactoring capabilities.
- **Component Recognition**: Automatically identifies and highlights Visualforce components, making it easier to work with complex page structures.
- **Expression Highlighting**: Highlights Visualforce expressions within HTML attributes and text nodes for better readability.
- **Code Folding**: Supports folding of code blocks for improved code management and navigation.

** LWC Mode
LWC mode is designed to support the development of Lightning Web Components (LWC) within Emacs. It offers:

- **Syntax Highlighting**: Provides tree-sitter-based syntax highlighting for JavaScript and HTML files, ensuring clear and distinct code visibility.
- **Code Completion**: Leverages company-mode for intelligent suggestions and completions, enhancing the development experience.
- **LSP Integration**: Integrates with both Eglot and LSP-bridge for language server features, offering enhanced code intelligence and navigation.
- **Component Recognition**: Automatically identifies and highlights LWC components, facilitating easier component management.
- **Expression Highlighting**: Highlights LWC template expressions within HTML attributes and text nodes for better readability.
- **Project Detection**: Automatically detects LWC files within Salesforce projects and applies appropriate settings.

** SOQL Mode
SOQL mode is designed to facilitate the development and execution of SOQL queries within Emacs. It offers:
- **Syntax Highlighting**: Utilizes tree-sitter for clear and distinct syntax highlighting of SOQL statements.
- **Auto-Completion**: Leverages company-mode to provide intelligent suggestions and completions for SOQL fields and keywords.
- **LSP Integration**: Supports both Eglot and LSP-bridge for language server features, enhancing code navigation and refactoring capabilities.
- **Query Execution**: Allows for the execution of SOQL queries with result visualization, making it easier to interact with Salesforce data.
- **Customization**: Offers customizable settings for LSP paths and completion types, allowing for a tailored development experience.
