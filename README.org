#+title: Salesforce Development Tools for Emacs

* Overview
A comprehensive Emacs package for Salesforce development, integrating Salesforce CLI with Emacs for a streamlined development workflow.

** Key Features
- Full Salesforce CLI integration
- Multi-org management and switching
- Metadata retrieve/deploy with diff support
- Data import/export with org-mode integration
- Apex code execution
- SOQL/SOSL query execution
- Consult integration for unified search
- Transient menus for complex operations

* Architecture

This package is part of a multi-package ecosystem for Salesforce development:

| Package               | Purpose                              | Repository                      |
|-----------------------+--------------------------------------+---------------------------------|
| salesforce-minor-mode | Core CLI integration (this package)  | [[https://github.com/tan-minh-nguyen/salesforce-minor-mode][salesforce-minor-mode]]           |
| apex-ts-mode          | Apex tree-sitter major mode          | [[https://github.com/tan-minh-nguyen/apex-ts-mode][apex-ts-mode]]                    |
| soql-ts-mode          | SOQL tree-sitter major mode          | [[https://github.com/tan-minh-nguyen/soql-ts-mode][soql-ts-mode]]                    |
| lwc-ts-mode           | LWC tree-sitter major mode           | [[https://github.com/tan-minh-nguyen/lwc-ts-mode][lwc-ts-mode]]                     |
| visualforce-ts-mode   | Visualforce tree-sitter major mode   | [[https://github.com/tan-minh-nguyen/visualforce-ts-mode][visualforce-ts-mode]]             |
| apex-ts-log           | Apex debug log tree-sitter mode      | [[https://github.com/tan-minh-nguyen/apex-ts-log][apex-ts-log]]                     |

The tree-sitter modes are standalone packages that work independently for syntax highlighting and editing. When combined with salesforce-minor-mode, they gain additional features like org-babel integration, CLI commands, and project management.

* Installation
** Dependencies
- [[https://developer.salesforce.com/docs/atlas.en-us.sfdx_setup.meta/sfdx_setup/sfdx_setup_install_cli.htm][Salesforce CLI]] (sf) v2.x+
- Emacs 27.1+
- [[https://tree-sitter.github.io/tree-sitter/][tree-sitter]] (for companion mode packages)

** Doom Emacs
#+begin_src emacs-lisp
;; packages.el
(package! salesforce-minor-mode
  :recipe (:host github
           :repo "tan-minh-nguyen/salesforce-minor-mode"
           :files ("*.el" "extensions/*.el")))

;; Optional: Install companion tree-sitter modes
(package! apex-ts-mode
  :recipe (:host github :repo "tan-minh-nguyen/apex-ts-mode"
           :files ("*.el" "extensions/*.el")))
(package! soql-ts-mode
  :recipe (:host github :repo "tan-minh-nguyen/soql-ts-mode"
           :files ("*.el" "extensions/*.el")))
(package! lwc-ts-mode
  :recipe (:host github :repo "tan-minh-nguyen/lwc-ts-mode"
           :files ("*.el")))
(package! visualforce-ts-mode
  :recipe (:host github :repo "tan-minh-nguyen/visualforce-ts-mode"
           :files ("*.el" "extensions/*.el")))
(package! apex-ts-log
  :recipe (:host github :repo "tan-minh-nguyen/apex-ts-log"
           :files ("*.el" "extensions/*.el")))

;; config.el
(use-package! salesforce-minor-mode
  :hook ((apex-ts-mode . salesforce-mode)
         (soql-ts-mode . salesforce-mode)
         (visualforce-ts-mode . salesforce-mode)
         (lwc-ts-mode . salesforce-mode)))
#+end_src

** Key Bindings
*** Main Prefix Keys
| Key   | Action                   |
|-------+--------------------------|
| =M-o o= | Org management menu      |
| =M-o r= | Run/Execute code menu    |
| =M-o R= | Resource management menu |
| =M-o A= | Authorize org            |

*** Org Management (=M-o o=)
| Key | Action               |
|-----+----------------------|
| =TAB= | Switch org           |
| =.=   | Open org in browser  |
| =r=   | Retrieve metadata    |
| =d=   | Deploy/push metadata |
| =p=   | Diff local vs remote |

*** Run/Execute Code (=M-o r=)
| Key | Action              |
|-----+---------------------|
| =q=   | Execute SOQL query  |
| =s=   | Execute SOSL search |
| =a=   | Execute Apex code   |

*** Resource Management (=M-o R=)
| Key | Action                     |
|-----+----------------------------|
| =c=   | Create Salesforce resource |
| =l=   | Clear log data             |

* Features
** Apex Development
- Execute Apex code against connected orgs
- Generate classes and triggers
- Org-babel support (via apex-ts-mode extensions):
  #+begin_src org
  ,#+begin_src apex :org myorg
    System.debug('Hello from Emacs!');
  ,#+end_src
  #+end_src

** SOQL Queries
- Query execution with results visualization
- Org-babel support (via soql-ts-mode extensions):
  #+begin_src org
  ,#+begin_src soql :org myorg :results table
  SELECT Id, Name FROM Account LIMIT 10
  ,#+end_src
  #+end_src

** Data Operations
- Bulk import/export
- Org-table integration
- CSV and JSON support

** Multi-Org Management
- Authenticate and switch between orgs
- List and filter orgs (sandbox, production, scratch)
- View org status and logs

** Project Management
- Initialize projects
- Deploy/retrieve metadata
- Compare local and remote changes
- Ediff integration for conflicts

* Companion Packages

Each tree-sitter mode package follows a similar structure:

#+begin_example
<mode>-ts-mode/
├── <mode>-ts-mode.el         # Main mode (standalone)
├── <mode>-*.el               # Supporting files
├── extensions/
│   ├── <mode>-ob.el          # Org-babel integration (requires salesforce-*)
│   ├── <mode>-lsp.el         # LSP integration
│   └── <mode>-*.el           # Other integrations
└── README.org
#+end_example

The =extensions/= folder in each mode package contains files that integrate with salesforce-minor-mode. These are loaded with soft requires and work optionally.
