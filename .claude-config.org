* Language Rules

** Emacs Lisp Rules

*** FILE ORDER (MANDATORY)
1. Header (~-*- lexical-binding: t; -*-~)
2. Requires
3. Defgroup + Defcustoms
4. Cl-defstruct
5. Defvar / Defvar-local
6. Functions
7. Provide + Footer

*** CODING RULES
- DRY: No duplicated logic, extract helpers
- Comments: Docstrings yes, inline only for "why"
- Messages: No debug output, minimal user messages
- Params: 0-2 positional OK, 3+ use ~cl-defun~ with ~&key~
- Size: <20 lines per function, single responsibility

*** TEMPLATE
#+begin_src emacs-lisp
;;; pkg.el --- Desc -*- lexical-binding: t; -*-
;;; Commentary:
;;; Code:
(require 'cl-lib)

(defgroup pkg nil "Group." :group 'tools :prefix "pkg-")
(defcustom pkg-opt "val" "Option." :type 'string :group 'pkg)

(cl-defstruct (pkg-item (:constructor pkg-item-create)) name value)

(defvar pkg--cache nil "Cache.")
(defvar-local pkg--state nil "State.")

(defun pkg--internal () "Internal." nil)
;;;###autoload
(defun pkg-command () "Command." (interactive) (pkg--internal))

(provide 'pkg)
;;; pkg.el ends here
#+end_src

*** ANTI-PATTERNS
- Mixed defvar/defcustom order
- Requires after code
- Struct after variables
- Missing lexical-binding
- God functions (100+ lines)
- Debug messages in production
- Too many positional params
